data.table und dplyr

Daten lesen

Für die Datenmengen mit denen wir uns hier beschäftigen sind die standard-tools (e.g. read.table) von R zu ineffizient. Abhilfe schafft fread aus dem Paket data.table:

require(data.table)
require(bit64)

data.location.raw <- fread('loc.tsv', colClasses=list(character=1:20))
data.participants <- fread('participants.tsv', colClasses=list(character=1:9))
data.demographics <- fread('demographics.tsv', colClasses=list(character=1:8))

# Einige Spalten sind inkonsistent benannt
data.demographics <- rename(data.demographics, ExternalId=ExternalID)

Daten lesen

data.table (II/II)

Um den Prozess weiter zu beschleunigen, können Spalten ausgeschlossen oder der Ladevorgang auf die ersten n Zeilen beschränkt werden:

data.usage <- fread(
  'ase.tsv',
  colClasses=list(
    character=1:8 # In diesem Fall enthalten alle Spalten Text
    )
  nrows=10000, # Wir beschränken uns auf die ersten 10K Zeilen
  drop='PackageName' # Redundante Spalte wird ausgeschlossen
  )

dplyr

Bestandteile: Verben

Verben beschreiben einzelne Analyseschritte

  • sample_n() und sample_frac() ziehen Stichproben
  • filter() und select() filtern Zeilen und Spalten nach spezifizierten Kriterien
  • mutate() und summarize() berechnen neue Spalten
  • group_by() gruppiert Einträge
  • join verbindet unterschiedliche Datensätze
  • …

dplyr

Bestandteile: Pipes

Ceci n'est pas une pipe

Aufeinanderfolgende Schritte werden mit %>% verknüpft:

require(dplyr)

# Berechnet die Gesamtnutzungszeit pro App
data.usage.summarized <- data.usage.raw %>%
  group_by(AppName) %>%
  summarize(
    n=n()
  )

dplyr

Verben: sample()

Der erste Schritt euerer Analyse sollte eine drastische Datenreduktion sein, im einfachsten Fall eine Zufallsstichprobe der Daten.

# ... mit einer festen Fallzahl
data.usage.subsample <- data.usage.raw %>%
  sample_n(10^3) # (*size* gibt die Größe der Stichprobe an)
# ... oder alternativ als Anteil der Daten
data.usage.subsample <- data.usage.raw %>%
  sample_frac(0.10) # (*size* gibt den Anteil an den Ausgangsdaten an)

data.usage.whatsapp <- data.usage.raw %>%

dplyr

Verben: select()

Eine weitere Möglichkeit zur Reduktion der Datenmenge ist die Beschränkung auf einzelne Spalten.

data.location <- data.location.raw %>%
  select(
    matches('UserHash'),
    ends_with('OnStart'),
    ends_with('OnEnd'),
    -starts_with('Altitude')#, ...
  )

dplyr

Verben: mutate()

Während wir bisher Daten weggeworfen haben, können wir mit mutate aus den bestehenden Daten neue generieren:

# Das Paket geosphere bietet funktionen um Umgang mit Koordinaten
require(geosphere)

data.location <- data.location %>%
  mutate(
    Distance=geosphere::distCosine(
      data.location %>% select(one_of(c('LongitudeOnStart', 'LatitudeOnStart'))),
      data.location %>% select(one_of(c('LongitudeOnEnd', 'LatitudeOnEnd')))
    )  
  )

dplyr

Verben: summarize()

Ganz ähnlich wie mutate funktioniert summarize, mit dem Unterschied dass es alle früheren Spalten verwirft.

data.usage  %>%
  summarize(
    count=n()  
  )

dplyr

Verben: group_by()

Der Befehl group_by unterteilt die Daten in Gruppen, auf die nachfolgende Befehle jeweils getrennt angewendet werden. 

data.apps.most_popular <- data.usage %>%
  group_by(AppName) %>% # Gruppiere nach App
  summarize(
    count=n()  
  ) %>%
  ungroup() %>% # Arbeite ohne Gruppierung weiter
  mutate(
    rank=rank(-count)  
  ) %>%
  filter(
    rank <= 10  
  )

dplyr

Verben: arrange()

Die Daten können mit arrange() sortiertert werden

data.apps.most_popular <- data.usage.raw %>%
  group_by(AppName) %>% # Gruppiere nach App
  summarize(
    count=n()  
  ) %>%
  ungroup() %>% # Arbeite ohne Gruppierung weiter
  mutate(
    rank=rank(-count)  
  ) %>%
  filter(
    rank <= 10  
  ) %>%
  arrange(desc(rank))

dplyr

Verben: *_join()

Anstelle des typischen R-Befehls merge bietet dplyr eine Gruppe von Befehlen zum (wesentlich schnelleren) Zusammenführen von Datensätzen.

data.usage %>%
  group_by(UserHash) %>%
  summarize(
    count=n()  
  ) %>%
  ungroup() %>%
  inner_join(data.participants, by="UserHash") %>%
  inner_join(data.demographics, by="ExternalId")

lubridate

Der Umgang mit Zeiten kann in R lästig sein. Mit lubridate wird das einfacher.

data.usage %>%
  mutate(
    AppStartTime_ts = lubridate::ymd_hms(AppStartTime_ts),
    AppEndTime_ts   = lubridate::ymd_hms(AppEndTime_ts),
    usageTime       = as.numeric(difftime(AppEndTime_ts,AppStartTime_ts,units = "mins")),
    month           = lubridate::month(AppEndTime_ts,label = T),
    day             = lubridate::wday(AppEndTime_ts,label = T)
    )
